## 关键决策

- mpv 优先于 ffplay/VLC，因为 mpv 提供稳定的 JSON IPC。
- 播放器优先级调整为 mpv → VLC oldrc → ffplay。
- VLC RC 仅绑定本机 TCP，命令严格串行并验证输入长度与控制字符。
- 播放器子进程由独立 Node 守护进程监视父级 IPC；正常退出解除守护，父进程硬终止则强制清理播放器进程树。
- 无 mpv 时继续使用现有重启式控制，避免破坏兼容性。
- IPC 使用每次播放唯一的 Windows named pipe，命令使用递增 request_id。
- 测试通过依赖注入模拟进程与 IPC，不要求开发机安装 mpv。

## 风险或阻塞

- 当前环境只有 ffplay，无法执行真实 mpv/VLC 音频验收；IPC/RC 命令、加载时序、代次过滤、连接超时和 socket 生命周期已由模拟测试覆盖。
- UI 进度仍使用本地平滑时钟，尚未订阅 mpv `time-pos` 做长期漂移校准。
